# デモシナリオ

## 前提
- PC画面共有（Discord）
- ローカルでBE/FE起動済み
- store_1（酒肴処 赤提灯 よりみち）にシードデータ投入済み（間違い3箇所、公開済み）

## 起動手順
```bash
# BE
cd backend && uv run uvicorn app.main:app --reload

# FE
cd frontend && bun dev
```

## シードデータ再投入（必要な場合）
```bash
cd backend
rm -f database.db
uv run python seed.py
```

---

## パート1: 管理者フロー

### 1. 店舗登録
- URL: http://localhost:5173/admin/register
- ※ デモではシード済みなのでスキップ可

### 2. 管理画面トップ（ベース画像一覧）
- URL: http://localhost:5173/admin/stores/1
- 生成済み画像カードが表示されている
- 「生成する」ボタン → 確認モーダル → 生成の流れを説明
- 画像カードクリック → 詳細モーダル → 公開/非公開の切り替え

### 3. 店舗設定
- URL: http://localhost:5173/admin/stores/1/settings
- ジャンル・店名・紹介文・メニュー説明・写真が入力済み
- 「保存する」→ トースト表示

### 4. QRコード表示
- URL: http://localhost:5173/admin/stores/1 → 右上「QRコード」ボタン
- 「テーブルに設置してお客様に読み取ってもらう」と説明

---

## パート2: お客さんフロー

### 5. ランディング画面
- URL: http://localhost:5173/play/1
- 店名・遊び方ガイドが表示
- 「スタート」をクリック

### 6. ゲームプレイ
- URL: http://localhost:5173/play/1/game
- 上下2枚の画像を見比べて間違いを探す
- クリックで間違いを指定（正解→緑丸、不正解→赤バツ）
- タイマー（1分10秒）＋プログレスバー

### 7. 結果画面
- URL: http://localhost:5173/play/1/result
- 全問正解 or タイムアップ
- 発見数・クリアタイム表示
- 「もう一度遊ぶ」で再プレイ可能

---

## トークポイント
- 飲食店の待ち時間を楽しく → 顧客体験向上
- 店舗側は登録するだけ、画像はAIが自動生成
- QRコードをテーブルに置くだけで導入完了
- 画像生成パイプライン: Groq API + セグメンテーション + インペイント

## 注意
- 画像生成をリアルタイムで回すとタイムリスク → シードデータを使う
- PC操作なのでピンチズームは見せられない → クリック操作でデモ
